### <a name="_u8xz25hbrgql"></a>**Метрики для оценки эффективности**
| **№** | **Метрики**                                | **Как поможет в оптимизации**                                      |
|:-----:|:-------------------------------------------|--------------------------------------------------------------------|
|   1   | Количество правильно протегированых данных | Доработка правил тегрования, выделение групп для тегов             |

|   3   | Частота использования тегов                | Решим нужно ли нам добавлять дополнительные теги для процессов     |

### <a name="_u8xz25hbrgql"></a>**Обеспечение scalability системы**

Для масштабирования процесса классификации данных можно разбить процесс на несколько этапов

* Определение группы для информации(например на основе формата: картинка текст и тд)
* Проставление тегов конфиденциальности
* Проставление тегов служебных (для архивирования дальнейшего и тд)

Для каждого этапа можно выделить отдельный слой и добавлять ресурсов ему, так же можно добавить кеширование правил в отдельном кеше (redis и тд)
Так же выделить отдельное хранилище для исторических данных (возможно при загрузке данных, например миграции уже сразу будут данные которые можно отправлять туда)
